1. Konsep Dasar Jaringan dan Pengalamatan üåê

Memahami fondasi dari sebuah layanan jaringan, mulai dari struktur komunikasi hingga sistem pengalamatannya.

1.1 Model Referensi Jaringan (OSI dan TCP/IP)

Jaringan bekerja berdasarkan lapisan-lapisan standar untuk memastikan komunikasi antar perangkat berjalan lancar, terlepas dari jenis perangkat kerasnya.

  • **Model OSI (Open Systems Interconnection):** Memiliki **7 lapisan** (Physical, Data Link, Network, Transport, Session, Presentation, Application). Ini adalah model teoritis.
  • **Model TCP/IP (Transmission Control Protocol/Internet Protocol):** Model yang digunakan secara praktis, terdiri dari **4 lapisan** (Network Access, Internet, Transport, Application).
  • **Fungsi Lapisan Kunci:** Lapisan **Network** (IP addressing), Lapisan **Transport** (TCP/UDP), Lapisan **Application** (Protokol layanan, seperti HTTP, FTP).

1.2 Pengalamatan IP (Internet Protocol)

Setiap perangkat dalam jaringan wajib memiliki alamat unik agar paket data bisa sampai ke tujuan yang benar.

  • **IP Versi 4 (IPv4):** Terdiri dari 32 bit, format *dotted decimal*. Pembagian menjadi kelas A, B, C, D, dan E.
  • **Subnetting:** Teknik untuk memecah *network address* menjadi sub-network yang lebih kecil dan efisien. Konsep **CIDR** (Classless Inter-Domain Routing).
  • **IP Versi 6 (IPv6):** Terdiri dari 128 bit, format *hexadecimal*. Solusi untuk keterbatasan alamat pada IPv4.

2. Implementasi Layanan Jaringan Esensial ‚öôÔ∏è

Layanan dasar yang wajib ada di setiap infrastruktur jaringan, baik lokal maupun internet.

2.1. Dynamic Host Configuration Protocol (DHCP) Server

DHCP adalah layanan yang bertugas mendistribusikan IP Address, Subnet Mask, Gateway, dan DNS Server secara otomatis kepada klien. Ini mengurangi beban administrasi jaringan.

  • **Konsep *Lease Time*:** Durasi sewa IP Address oleh klien.
  • **DHCP *Scope*:** Batasan range IP Address yang akan dibagikan.
  • **DHCP *Exclusion*:** Daftar IP Address yang tidak boleh dibagikan.

2.2. Domain Name System (DNS) Server

Fungsi utamanya adalah sebagai "penerjemah" dari Nama Domain (contoh: google.com) menjadi IP Address (contoh: 172.217.16.14). Tanpa DNS, kita harus mengingat IP setiap situs.

  • **Zonasi DNS:** *Forward Look-up Zone* (Nama ke IP) dan *Reverse Look-up Zone* (IP ke Nama).
  • **Record Penting:** A record (host address), CNAME record (alias), MX record (mail exchanger).

2.3. Web Server (HTTP/HTTPS)

Layanan yang menggunakan protokol **HTTP (Hypertext Transfer Protocol)** untuk mengirimkan halaman web ke browser klien. Umumnya menggunakan perangkat lunak seperti Apache, Nginx, atau IIS.

  • **Virtual Host:** Meng-hosting lebih dari satu domain dalam satu server fisik.
  • **HTTPS:** Versi aman dari HTTP, menggunakan protokol **SSL/TLS** untuk enkripsi data.

3. Teknologi Voice over Internet Protocol (VoIP) üìû

VoIP adalah teknologi yang memungkinkan transmisi suara (panggilan telepon) melalui jaringan Internet Protocol (IP). Ini adalah salah satu layanan jaringan paling modern.

3.1. Arsitektur dan Protokol VoIP

Komunikasi VoIP memerlukan *signaling protocol* untuk memulai, memodifikasi, dan mengakhiri sesi panggilan, serta *media protocol* untuk membawa data suara.

  • **Signaling Protocol Utama:** **SIP (Session Initiation Protocol)** yang ringan dan populer.
  • **Media Protocol:** **RTP (Real-time Transport Protocol)** untuk membawa paket data suara.
  • **Codec:** Algoritma yang digunakan untuk mengompresi dan mendekompresi sinyal suara (misalnya G.711, G.729).

3.2. Konfigurasi Server PBX (Private Branch Exchange)

PBX server (seperti Asterisk atau FreePBX) berfungsi sebagai pusat kontrol panggilan telepon di dalam suatu organisasi.

  • **Ekstensi:** Nomor unik untuk setiap pengguna/perangkat di dalam jaringan PBX.
  • **Dial Plan:** Aturan dan skema penomoran yang menentukan bagaimana panggilan diarahkan.

4. Manajemen dan Keamanan Jaringan üõ°Ô∏è

Layanan yang berfokus pada kontrol lalu lintas data dan proteksi jaringan dari ancaman luar.

4.1. Router dan NAT (Network Address Translation)

Router berfungsi menghubungkan dua network atau lebih. NAT adalah layanan yang memungkinkan banyak perangkat di jaringan lokal berbagi satu IP publik untuk akses internet.

4.2. Firewall dan Filtering

Firewall berfungsi sebagai pagar pembatas antara jaringan internal dan eksternal. Firewall dapat berbasis perangkat keras (Hardware) atau perangkat lunak (Software).

  • **Konsep *Chain* dan *Rule*:** Urutan aturan (rules) dalam sebuah firewall (Input, Output, Forward).
  • **Filtering:** Memblokir atau mengizinkan lalu lintas berdasarkan IP Address, Port, atau Protokol.

4.3. Bandwidth Management

Mengelola dan membatasi kecepatan transfer data setiap pengguna atau layanan agar tidak ada satu pengguna pun yang memonopoli bandwidth.

  • **Queue:** Mekanisme antrian untuk mengontrol paket data. (Misalnya, Simple Queue atau Queue Tree).
  • **Burst:** Pemberian kecepatan tinggi sesaat untuk meningkatkan pengalaman pengguna.
